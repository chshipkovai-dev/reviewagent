## Текущая проблема продукта

ReviewAgent сейчас — **инструмент, а не система.**

Пользователь должен сам:
- Помнить что нужно зайти
- Вручную скопировать отзывы
- Сгенерировать ответы
- Вручную скопировать обратно в Google

Нет уведомлений. Нет мониторинга. Нет автоматизации.

**Результат:** пользователи пользуются первую неделю, потом забывают → высокий churn → продукт не растёт.

## Критические улучшения (по приоритету)

### 🔴 Приоритет 1 — Google Business Profile API

**Что это меняет:** превращает инструмент в систему.

Как должно работать:
1. Пользователь один раз подключает Google Business аккаунт (OAuth)
2. Система автоматически мониторит новые отзывы
3. Приходит уведомление: "У вас 3 новых отзыва — нажмите чтобы ответить"
4. Заходишь → видишь готовые AI-ответы → нажимаешь "Опубликовать в Google"
5. Опционально: автоответ для типовых позитивных отзывов (5★ без текста)

**Почему это главное:** пользователь платит не за генерацию текста — он платит чтобы не думать об отзывах вообще. Без Google API продукт этого не даёт.

---

### 🔴 Приоритет 2 — Mapy.cz интеграция

Чешские рестораны и салоны получают отзывы **в первую очередь на Mapy.cz**, не Google.

Без Mapy.cz половина чешских клиентов скажет: "Мне это не подходит."

SUPPOO покрывает Mapy.cz. Нам нужно тоже.

Дополнительно рассмотреть: Heureka.cz, Zboží.cz (для e-commerce клиентов)

---

### 🟡 Приоритет 3 — Dashboard вместо пустой страницы

После логина пользователь должен видеть панель управления, не форму.

```
┌─────────────────────────────────────────────────────┐
│  Новых отзывов сегодня: 3    Отвечено: 87%    ★4.6  │
├──────────────────┬──────────────────────────────────┤
│  Требуют ответа  │  Последние ответы                │
│  ● 2-star от Jan │  ✓ Tomáš — ответили 2ч назад     │
│  ● 3-star от Mar │  ✓ Jana — ответили вчера          │
└──────────────────┴──────────────────────────────────┘
```

Метрики на dashboard:
- Количество новых отзывов
- % отвеченных отзывов
- Средняя оценка за последние 30 дней (тренд ↑↓)
- История ответов

---

### 🟡 Приоритет 4 — Email дайджест

Каждый понедельник утром автоматически:
> "У вашего бизнеса 5 новых отзывов за неделю. Средняя оценка 4.3 ★. Нажмите чтобы ответить."

Без этого пользователи забывают про продукт → churn.

Стек: n8n + Resend (или Supabase Edge Functions + Resend)

---

### 🟢 Приоритет 5 — Multi-location поддержка

Один аккаунт → несколько бизнесов/локаций.

Для кого: сети ресторанов, franchise, владельцы нескольких салонов.

Это открывает Enterprise тариф и значительно повышает средний чек.

---

### 🟢 Приоритет 6 — Sentiment analytics

Показывать владельцу тренды:
- Что клиенты хвалят чаще всего
- На что жалуются
- Как меняется рейтинг со временем

Это превращает продукт из "отвечалки" в инструмент управления репутацией.

## Дизайн — что улучшить

### 1. Hero на лендинге — показать чешский пример
Не абстрактный английский отзыв. Конкретный чешский:

В секции "Работает с..." рядом с Google — логотип Mapy.cz.
Это сигнал что продукт сделан для Czech рынка.

### 3. Social proof
Три testimonial сейчас выглядят как фиктивные американские отзывы.
Убрать до появления реальных чешских клиентов.
Заменить на цифры: "73% клиентов ожидают ответа" и т.д.
